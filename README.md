# LINUXDO ReadBoost
[![GPLv3 License](https://img.shields.io/badge/License-GPLv3-blue.svg)](LICENSE)

[LINUXDO ReadBoost](https://greasyfork.org/scripts/519843-linuxdo-readboost)，一款润物细无声的脚本，解放双手，自动化刷取 LINUXDO 论坛的已读帖数
开源、温和，支持自定义参数，理论支持所有使用 Discourse 程序的论坛
⚠️：使用第三方脚本，存在失效、封号等潜在的风险，作为一款完全开源的脚本，支持使用者自行审核，开发者不对造成的后果负责

## 如何使用

1. 安装脚本（Tampermonkey / Violentmonkey）
2. 打开任意 LINUXDO 论坛的帖子页面
3. 在弹窗中同意使用许可
4. 在页面右上角找到 ReadBoost 的设置菜单和指示栏
5. 点击「设置」调整参数或直接点击「手动开始」
6. 建议在设置中勾选「自动运行」并保存，下次刷新自动生效

## 安全性

完全免费、开源，使用者可以自由查看并判断风险，开发者不对任何风险负责，也不对任何功能承诺，使用者需要自行承担使用该脚本的后果

## 设置参数

| 参数 | 说明 | 默认值 | 建议范围 |
| --- | --- | --- | --- |
| 基础延迟 | 每次请求发起的间隔(ms) | 2000 | 500-10000 |
| 随机延迟范围 | 为基础延迟添加随机波动(ms) | 300 | 0-3000 |
| 最小每次请求阅读量 | 每批最少包含的回复数 | 8 | 1-50 |
| 最大每次请求阅读量 | 每批最多包含的回复数 | 20 | 1-100 |
| 最小阅读时间 | 每个回帖的最小模拟阅读时间(ms) | 800 | 100-10000 |
| 最大阅读时间 | 每个回帖的最大模拟阅读时间(ms) | 3000 | 100-30000 |

⚠️ 建议保持默认设置。如需提高速度，优先调大「每次请求阅读量」而非降低延迟，减少对服务器的影响。

## 相比原版的改进 (v1.3)

- **GPLv3 许可** — 从 MIT 切换至 GPLv3，使用更严格的 copyleft 保护

### v1.2

- **多论坛支持** — 使用 `location.origin` 动态获取 API 地址，新增 nodeloc.com / idcflare.com / meta.discourse.org 等 @match 规则
- **停止按钮** — 运行中可随时点击「停止」中断
- **暗色模式** — 设置弹窗自动跟随 Discourse 主题变量（`--secondary`, `--primary` 等）
- **输入校验** — 保存时自动 clamp 参数到合法范围，防止 NaN 或极端值
- **DOM 就绪等待** — 等待 DOMContentLoaded 后再操作，避免 SPA 竞态
- **后置延迟优化** — 最后一批请求后不再无谓等待
- **GM_addStyle** — 注入的样式跟随论坛主题变量，不再硬编码
- **元数据完善** — 添加 @icon、@updateURL/@downloadURL、@grant 声明
